PROMPT — Opção A: sem cópia/lifecycle para o modo prefixo JSONL
============================================================================

Opção A — sem cópia/lifecycle para o modo prefixo JSONL. Justificativa: são
artefatos intermediários gerados pela fase 1, não uploads de usuário; a
idempotência por combo no DynamoDB já protege contra reprocessamento.

Três ressalvas na implementação:

1. No branch JSONL-prefixo da agnóstica, PULE os estados
   GerarKeyProcessamento/CopiarParaProcessamento/DeletarOriginal e o
   MoverParaConcluidos/Falhas — o Map lê direto do prefixo original. O ciclo
   completo continua valendo para os modos de arquivo único
   (CSV/JSON/PARQUET), incluindo o pedido.csv da fase 1.

2. Documente no README da agnóstica essa diferença de comportamento entre
   os dois modos (arquivo único = com lifecycle; prefixo = sem).

3. Para não acumular lixo no bucket: registre como débito/sugestão uma S3
   Lifecycle Rule de expiração no prefixo
   disponibilidades/*/itens-extracao/ (ex.: 30 dias) — não implemente
   agora, só documente.
